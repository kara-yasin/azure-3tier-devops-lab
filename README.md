# Azure 3-Tier Infrastructure as Code with Pulumi & Python

This repository contains a modern, production-ready 3-tier network architecture and virtual machine infrastructure on Microsoft Azure, fully automated using **Pulumi (Python) and the Azure Native provider**.

## 📐 Architecture Overview
The project provisions a highly secure, logically isolated, and segmented network topology following enterprise cloud architecture principles:

* **Resource Group:** Centralized lifecycle management layer (`rg-3tier-devops-lab`).
* **Virtual Network (VNet):** `10.0.0.0/16` address space serving as the backbone.
* **3-Tier Subnet Segmentation:**
  * 🌐 `snet-web` (`10.0.1.0/24`) - Frontend tier for public-facing load balancers/proxies.
  * ⚙️ `snet-app` (`10.0.2.0/24`) - Isolated Application/Backend API tier (No Public IP).
  * 🗄️ `snet-db` (`10.0.3.0/24`) - Heavily restricted Database tier.
* **Compute Layer:** A cost-efficient Ubuntu Server VM (`Standard_B1s`) provisioned within the secure `snet-app` tier, utilizing custom network interfaces (NICs) and secure SSH key authentication.

## 🛠️ Tech Stack & DevOps Concepts
* **Infrastructure as Code (IaC):** Pulumi Python SDK (State managed securely via Pulumi Cloud).
* **Cloud Provider:** Microsoft Azure (`pulumi-azure-native`).
* **Core Concepts:** Network segmentation, secure micro-segmentation, dynamic private IP allocation, dependency tracking in IaC.

## 🚀 Quick Start & Deployment

### Prerequisites
* Active Azure Subscription & Azure CLI installed (`az login`).
* Pulumi CLI configured on your local machine.
* Python 3.10+ environment.

### 1. Clone & Setup
```bash
git clone [https://github.com/kara-yasin/azure-3tier-devops-lab.git](https://github.com/kara-yasin/azure-3tier-devops-lab.git)
cd azure-3tier-devops-lab
````
### 2. Infrastructure Deployment 
Initialize Pulumi and deploy your actual cloud infrastructure with a single command:
```bash
pulumi up
```
Review the preview generated by Pulumi, select yes, and watch the infrastructure orchestrate in real-time.

### 3.Project Structure
.
├── __main__.py          # Core IaC script defining the 3-tier Azure infrastructure
├── app.py               # Lightweight Python API server to be deployed on the App Tier VM
├── Pulumi.yaml          # Project metadata and runtime settings
├── Pulumi.dev.yaml      # Environment/Stack configuration (Region: westeurope)
└── requirements.txt     # Python dependencies (Pulumi SDK & Azure Native provider)

### 4. Outputs & Verification
Upon successful deployment, Pulumi exports the following key variables for upstream automation or CI/CD pipelines:

resource_group_name

vnet_name

To fetch outputs programmatically:
```bash
pulumi stack output vnet_name
```
